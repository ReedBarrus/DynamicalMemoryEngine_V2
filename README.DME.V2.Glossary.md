# README.DME.V2.Glossary.md

# DME V2 — Glossary

## 1. Purpose

This glossary defines the canonical vocabulary for DME V2.

It exists to prevent semantic drift across:

    - Architecture
    - Runtime Map
    - RIFT theory layers
    - runtime contracts
    - schemas
    - proofs
    - governance
    - projection
    - retention
    - agency

This glossary is implementation-facing.

Terms defined here are not rhetorical.

They are runtime contract terms.

---

## 2. Root Boundary

DME V2 uses the following boundary chain:

    source ≠ structure
    structure ≠ distinction
    distinction ≠ identity
    activity ≠ authority
    salience ≠ proof
    mapping ≠ mutation
    projection ≠ source
    retention ≠ identity
    agency ≠ sovereignty

Primary authority chain:

    possible ≠ admissible
    admissible ≠ proven
    proven ≠ authorized
    authorized ≠ permanent

Final operational boundary:

    runtime may reveal what can become structure;
    admissibility decides what may proceed;
    proof tests what survives;
    governance decides what may matter.

# A

## Activation Energy

    Runtime influence capacity assigned to an active Distinction-State.

    Activation energy contributes to whether a state may perturb, attract, stabilize, or decay.

    It is not authority.

    Typical state-vector position:

        Sᵢ = [ρ, E, H, K, φ, A, r]

    Where E = activation energy.

## Active Candidate

    A runtime structure currently admitted into the Runtime Manifold but not proven or authorized.

    Usually refers to a Distinction-State.

    Authority ceiling:

        active runtime candidate
        Active Distinction-State

    A Distinction-State currently hosted inside the Runtime Manifold.

    Contains:

        distinction region
        local frame
        state vector
        attention budget
        lineage ref
        uncertainty
        authority ceiling

    An active Distinction-State is not identity.

## Admissibility

    The local field of permitted transition.

    Admissibility answers:

        What may happen next?

    Formal:

        Admissible(s → s') iff:
            transition remains inside declared constraint envelope

    Expanded:

        Admissible(s → s') iff:
            locality_valid
            AND lineage_valid
            AND uncertainty_within_mode
            AND budget_available
            AND distinguishability_supported
            AND constraints_compatible
            AND temporal_order_preserved
            AND projection_boundary_respected
            AND proof_preconditions_met_if_required
            AND authority_ceiling_not_exceeded

    Admissibility is not authority.

## Admissibility Axis

    One dimension used to evaluate whether a runtime transition may proceed.

    Canonical axes:

        locality_validity
        lineage_validity
        uncertainty_bound
        budget_cost_bound
        salience_threshold
        distinguishability_support
        constraint_compatibility
        temporal_validity
        projection_boundary
        proof_precondition

    These axes define runtime permission geometry, not identity.

## Admissibility Decision

    A recorded result of evaluating a candidate transition against an admissibility profile.

    Possible outputs:

        allow
        allow_limited
        defer
        reduce_resolution
        route_to_index
        route_to_projection
        route_to_proof
        quarantine
        reject

    Every AdmissibilityDecision must declare an output ceiling.

## Admissibility Field

    The runtime constraint field that determines which activations, interactions, transitions, projections, proof entries, deferrals, and candidate emissions are permitted under current conditions.

    It operationalizes Admissibility Conservation.

    It governs runtime possibility.

    It does not grant consequence.

## Admissibility Mode

    A named runtime operating regime with threshold profiles, permitted outputs, blocked outputs, and failure policies.

    Canonical modes:

        exploratory
        diagnostic
        exploitative
        constrained
        proof_seeking
        recovery
        quarantine
        projection_only
        agent_limited
        governance_ready

    Modes are not authority states.

## Admissibility Profile

    A scoped configuration of admissibility mode, axes, thresholds, permitted outputs, blocked outputs, authority ceiling, and failure policy.

    Profiles may be configured by governance.

    Changing admissibility profiles is governance-relevant because it changes future runtime transition possibility.

## Agency

    The controlled ability of a projection-bound actor to read, propose, simulate, request proof, request governance, or execute scoped tools under explicit authority and admissibility constraints.

    Agency is not sovereignty.

## Agent

    A projection-bound actor operating under scoped authority and an explicit admissibility profile.

    An agent may:

        read scoped projections
        query indexes
        simulate branches
        emit proposals
        request proof
        request governance
        execute scoped tools if authorized

    An agent may not:

        self-promote authority
        write ledger truth directly
        mutate policy without governance
        erase failures
        hide uncertainty
        bypass projection limits
        emit training data without authorization

## AgentAuthority

    A governed record of what an agent may observe, request, simulate, propose, or execute.

    AgentAuthority must declare:

        projection scope
        admissibility mode
        allowed reads
        allowed proposals
        allowed simulations
        allowed tools
        proof requirements
        human review triggers
        revocation conditions
        expiry

    Authority ceiling:

        scoped actor permission

## Alignment

    One of the six invariant axes.

    Alignment measures ordering, phase, correspondence, sequence relation, or structural agreement.

    It is a comparison axis, not semantic agreement.

## Architecture

    The root identity surface of DME V2.

    Architecture defines:

        what DME V2 is.

    RuntimeMap defines:

        how DME V2 runs.

## Artifact

    A bounded structure recorded or emitted by the system.

    Artifacts may include:

        payload refs
        structural objects
        charts
        features
        distinction regions
        proof traces
        proof objects
        governance decisions
        projections
        retention records
        agent authority records

    Artifact does not imply proof or authority.

## Attention

    Bounded energy allocation over distinguishability.

    Attention determines which regions are sampled, amplified, processed, perturbed, or selected for further runtime action.

    Attention is not authority.

## Attention Budget

    The bounded runtime allocation assigned to active Distinction-States.

    Budget constrains:

        activation
        interaction
        proof routing
        branch projection
        runtime exploration
        field processing

    No active state may consume unbounded attention.

## Attention-Energy

    The bounded runtime resource that determines which Distinction-States are processed, perturbed, related, stabilized, or allowed to decay.

    It is a runtime analogue for:

        activation
        salience
        influence
        uncertainty pressure
        processing priority
        field deformation

    It is not literal heat.

    It is not authority.

## Authority

    A bounded, dynamic constraint field over permitted consequence.

    Authority answers:

        What may this structure affect?

        Authority does not mean truth.

        Authority governs consequence.

    Admissibility governs transition possibility.

## Authority Boundary

    The rule that authority cannot be inferred from activity, salience, recurrence, projection, proof, or agent confidence.

    Authority requires governance.

    Boundary:

        proof creates eligibility;
        governance grants consequence.

## Authority Ceiling

    The maximum claim or consequence a structure is allowed to carry.

    Examples:

        source material only
        measurement only
        candidate distinction
        active runtime candidate
        relation candidate
        proof evidence
        authority eligibility
        scoped consequence decision
        bounded view
        retention posture
        scoped actor permission

    Every object must respect its authority ceiling.

## Authority Field

    The governance constraint overlay over permitted consequence.

    It contains:

        authority vectors
        policies
        admissibility profiles
        proof dependencies
        projection boundaries
        agent boundaries
        risk gradients
        revocation conditions
        monitoring observables
        ledger refs

    The Authority Field is not the Runtime Manifold.

## Authority Vector

    A multidimensional authority representation.

    Canonical dimensions:

        epistemic_authority
        operational_authority
        mutation_authority
        exposure_authority
        retention_authority
        agent_authority
        admissibility_configuration_authority

    No dimension automatically grants another.

# B

## Basin

    A topology candidate formed when multiple trajectories or relations converge into a stable region.

    A basin is not static clustering.

    It is convergence behavior under relational dynamics.

    Authority ceiling:

        topology candidate

## Basis

    The representational frame used to expose or measure structure.

    Examples:

        time domain
        frequency domain
        token sequence
        syntax tree
        JSON graph
        embedding projection
        execution trace graph

    Every basis introduces bias.

## Basis Bias

    The structural effect introduced by representing a source through a specific basis.

    Bias is not necessarily error.

    Bias must be declared and carried forward.

## Basis Contribution Map

    A record of how much each chart, transform, or representational basis contributed to a distinction, uncertainty estimate, or invariant projection.

    Used to prevent hidden single-basis authority.

## Behavioral Attractor

    A recurring decision/transition pattern that repeatedly draws execution into similar paths.

    Behavioral attractors are mapped structures.

    They are not intent.

    They are not policy.

    They are not mutation authority.

## Behavioral Observable

    A T3 measurement of execution traces, agent/tool workflows, task dynamics, or decision processes.

    Examples:

        decision_point_count
        branching_factor
        tool_sequence
        tool_failure_rate
        repair_path
        constraint_violation_count
        drift_from_task_spec
        convergence_to_task_spec
        runtime_cost
        latency
        policy_friction

    Behavioral observables are measurements, not judgments.

## Behavioral Proof

    A proof type that tests whether a behavioral topology, decision path, tool order, repair strategy, or execution mutation preserves task identity and improves stability under perturbation or counterfactual replay.

    Behavioral proof does not authorize mutation.

## Branch

    A candidate transition path from a decision node.

    Branches may be:

        realized
        rejected
        counterfactual
        failed
        repaired
        deferred
        quarantined

    Branches are part of decision topology.

## Branch Collapse

    A failure mode where rejected, failed, or counterfactual branches are erased, hiding the actual decision structure.

## Branch Inflation

    A failure mode where too many speculative branches are generated, overwhelming budget and reducing runtime legibility.

## Budget

    A finite runtime resource boundary.

    May refer to:

        attention budget
        activation budget
        compute budget
        proof budget
        branch budget
        projection budget
        agent/tool cost budget

    Capacity is conserved.

    No runtime process may assume infinite budget.

# C
## Candidate

    A bounded structure proposed for further processing.

    Candidate types include:

        distinction candidate
        relation candidate
        topology candidate
        proof candidate
        governance candidate
        mutation candidate
        projection candidate
        retention candidate
        agent authority candidate

    Candidate does not imply proof.

## Canon

    A future governed status for structures that have passed stronger validation and governance.

    Door 1 should not attempt full canon formation.

    Canon is not defined as a root runtime primitive in current DME V2.

## Capacity Conservation

    The invariant that active bindings, states, relations, proofs, projections, and authorities are finite.

    When capacity is exceeded, the system must:

        defer
        summarize
        index
        compress
        decay
        release
        quarantine
        request proof

    It may not silently continue as if capacity were infinite.

## Causal Signature

    A retained trace of causal contribution, lineage, transition effect, or influence pattern sufficient to support future reconstruction, replay, proof, audit, or governance review.

    Causal signatures belong primarily to Retention, Ledger, Decision Topology, Proof, and Governance.

    A causal signature is not proof by itself.

## Chart

    A declared coordinate representation of structure under a specific basis.

    Examples:

        waveform
        frequency spectrum
        token sequence
        syntax tree
        JSON graph
        embedding projection
        execution trace graph

    Authority ceiling:

        basis-local representation

## Chart Transition

    A transform between coordinate representations.

    Examples:

        time → frequency
        raw text → syntax tree
        JSON object → graph
        trace log → execution graph

    Every chart transition must declare:

        basis
        scale
        metric
        domain
        uncertainty
        bias
        loss

## Coherence

    A condition where structure, relation, or behavior preserves enough alignment, compatibility, or invariant continuity under declared context.

    Coherence is not proof unless tested.

    Coherence is not authority.

## Commit

    A T2 phase operator.

    Commit stabilizes a structure, transition, proof, or record into a persistence boundary under declared scope.

    Commit does not imply semantic truth or unlimited authority.

## Composition

    One of the six invariant axes.

    Composition measures what structure is made of.

    Examples:

        distribution contents
        component set
        token mixture
        spectral composition
        graph element distribution

## Compression

    A reduction of structure into a smaller representation.

    Compression must declare:

        preserved axes
        omitted axes
        loss profile
        uncertainty
        reconstruction bounds
        authority ceiling

    No free compression is allowed.

## Constraint

    A limit on possible states, transitions, influence, authority, or projection.

    Constraints define:

        boundaries
        valid transformations
        admissible influence
        possible trajectories
        system limits

    No field exists without constraint.

## Constraint Compatibility

    An admissibility axis that determines whether a transition violates structural, runtime, proof, projection, agent, or governance constraints.

## Constraint Strength

    A state-vector quantity measuring how sharply bounded or resistant a Distinction-State is.

    Typical symbol:

        K

    High constraint strength can stabilize boundaries, but does not prove identity.

## Context

    The declared conditions under which observation, comparison, proof, projection, authority, or governance occurs.

    Context may include:

        lens
        scope
        domain
        scale
        metric
        weights
        source family
        runtime mode
        admissibility profile
        authority ceiling
        declared use

    No proof is context-free.

## Convergence

    One of the six invariant axes.

    Convergence measures agreement across paths, bases, derivations, traces, or independent supports.

    Convergence is not consensus by popularity.

    It is structural agreement across declared paths.

## Convergence Region

    A behavioral or relational region where multiple trajectories converge toward stable task completion, structural coherence, repair success, or identity support.

    Authority ceiling:

        convergence candidate

## Counterfactual Branch

    An unrealized but admissible candidate transition path used for decision topology mapping or proof challenge.

    Counterfactual branches are not actions.

    They are candidate futures.

## Cross-Basis Analysis

    Comparison of multiple chart representations of the same source-local structure.

    Produces:

        invariant agreement
        basis disagreement
        uncertainty gradients
        bias maps
        candidate distinction regions

    Cross-basis analysis is required to avoid single-basis authority.

## Curvature

    A field observable measuring how much a state deforms local relation geometry.

    Useful for detecting:

        attractor formation
        context reorganization
        anomaly influence
        trajectory bending
        basin formation

    Curvature is not proof.

# D

## Data

    Raw material or represented content prior to distinction under observer/lens/constraint.

    Data is not information by default.

    Information is distinguishable structure relative to observer, lens, and constraint.

## Decision Node

    A detected branching condition within distinction-space recursion.

    Formal:

        DecisionNode(Dᵢ) iff:
            |T_admissible(Dᵢ)| ≥ 2
            AND distinguishable(T₁...Tₙ) ≥ τ_branch
            AND H(Dᵢ) ≥ H_min
            AND realized_transition exists or is being selected

    A decision node is not intent, agency, semantic choice, proof, or authority.

    It is:

        a region of constrained possibility where multiple futures are available and one transition becomes realized.

## Decision Topology

    A graph/field of detected decision nodes connected by admissible transitions.

    Contains:

        DecisionNodes
        TransitionEdges
        RealizedTrajectories
        RejectedBranches
        FailureSurfaces
        RepairPaths
        DriftRegions
        ConvergenceRegions
        BehavioralAttractors

    Decision topology maps behavior.

    It does not authorize behavior change.

## Decision Topology Mapping

    The applied projection that converts execution traces into branchable relational structure over decision space.

    Core principle:

    Mapping says: here is the shape of behavior.
    Proof says: this behavior survives challenge.
    Governance says: this behavior may alter future execution.

## Decay

    A runtime process where inactive, unstable, low-salience, or unsupported states lose activation or attention.

    Decay must preserve trace where consequential.

    Decay is not deletion.

## Defer

    A safe routing outcome where processing pauses while retaining trace, uncertainty, and future re-entry possibility.

    T2 macro:

        Defer = Release + Record + optional Project(index/support handle)
        Derived Artifact

    A structure produced from prior material through declared operations.

    Examples:

        StructuralObject
        Chart
        Feature
        DistinctionRegion
        EventEnvelope
        RelationCandidate
        Projection
        ProofObject

    Derived artifacts must preserve lineage.

## Detector

    A T3 recognition function over observables.

    Detectors answer:

        Has a pattern crossed from noise into something worth tracking?

        A detector does not prove or authorize the structure it detects.

    Authority ceiling:

        detection only

## Diagnostic Mode

    An admissibility mode used for inspecting ambiguity, failure, instability, drift, or anomaly.

    Allows high-uncertainty probing.

    Blocks mutation and durable promotion.

## Difference

    Variation between states.

    Difference is not yet distinction.

    Difference becomes distinction only when detectable, bounded, supported, admissible, and provenance-linked.

## Distinguishability

    The capacity for a difference to remain separable under declared observer, lens, metric, scale, constraint, and uncertainty bounds.

    Distinguishability is conserved across lawful transformation unless loss is declared.

## Distinguishability Support

    An admissibility axis that checks whether a transition preserves or improves resolvable structure.

    Failure modes include:

        distinction collapse
        over-merge
        underresolved region
        semantic projection
        unsupported transition

## Distinction

    A bounded, observer-relative difference satisfying support and admissibility conditions.

    A distinction is not:

        raw feature
        semantic label
        identity
        proof
        authority

    A distinction is the first runtime-selectable unit of cognition/runtime.

## DistinctionEvidencePacket

    The handoff object from Structural Admission into Distinction Formation.

    Contains:

        source refs
        structural refs
        chart refs
        feature refs
        basis declarations
        domain map
        metric map
        scale map
        uncertainty map
        bias map
        candidate regions

    Authority ceiling:

        evidence packet only

## Distinction Field

    The set of detectable differences under a given observer, context, lens, and constraint.

    A distinction field is:

        multi-scale
        multi-basis
        attention-weighted
        uncertainty-aware

## Distinction Formation

    The runtime layer that converts cross-basis evidence into bounded DistinctionRegions.

    It performs:

        cross-basis comparison
        agreement/disagreement mapping
        uncertainty declaration
        bias declaration
        invariant-axis projection
        distinction support evaluation

## Distinction Geometry

A multi-chart, multi-scale field of distinguishable structure derived from cross-basis analysis of source-local topology.

It replaces EventGeometry as the primary runtime geometry.

Distinction Geometry produces:

invariant vectors
uncertainty fields
bias maps
candidate regions

It does not produce proof, authority, or meaning.

## Distinction Operator

    The operator that converts DistinctionEvidencePackets into DistinctionRegions.

    It answers:

        Which patterns are sufficiently separable, supported, and uncertainty-bounded to become active runtime structures?

## DistinctionRegion

    A bounded region of structure whose separation, support, lineage, admissibility, and uncertainty conditions make it runtime-selectable.

    Authority ceiling:

        candidate distinction
        
## Distinction-State

    The primary active runtime unit of DME V2.

    A Distinction-State is a DistinctionRegion admitted into the Runtime Manifold with:

        local frame
        state vector
        attention budget
        lineage ref
        uncertainty
        authority ceiling

    A Distinction-State is not identity.

## Domain

    The space where values exist.

    Examples:

        time-amplitude
        frequency
        token sequence
        tree
        graph
        execution trace

    Domain is distinct from metric.

## Drift Region

    A behavioral or relational region where structure or behavior moves away from task identity, proof constraints, declared scope, or stable invariance.

    Authority ceiling:

        drift candidate

# E

## Echo

    A T2 phase operator.

    Echo reintroduces prior trace into current admissible processing as influence, support, recurrence, or constraint.

    Echo does not make the past present.

    Echo does not grant authority by recurrence alone.

## Emission

    Source-native material encountered or generated by the system.

    Examples:

        audio array
        text stream
        JSON object
        OS event
        tool call
        model call
        file diff
        test result
        log
        sensor reading

## Encoding

The format and representation of a native emission.

Encoding must be preserved at Native Emission Capture.

Incorrect encoding interpretation is a source failure.

## Energy

    The capacity for structure to change, propagate, or interact.

    Energy enables:

        activation
        transformation
        attention
        interference
        feedback
        persistence maintenance

    Energy is not authority.

## Epistemic Authority

    An authority-vector dimension measuring how strongly a structure may be treated as validated within declared scope.

    Examples:

        unvalidated
        support_only
        bounded_identity_candidate
        validated_identity
        contested_identity
        revoked_identity

    Epistemic authority does not grant mutation by itself.

## Event

    A derived or projected occurrence-like artifact.

    In DME V2, event is no longer a primary runtime primitive.

    Events may be used for UI, query, compatibility, or projection purposes.

    Runtime operates on Distinction-States.

## EventEnvelope

    A resolved artifact or projection-ready bounded distinction structure.

    Allowed:

        UI display
        query result
        resolved ledger artifact
        projection artifact
        legacy compatibility
        bounded occurrence reporting

    Disallowed:

        primary runtime primitive
        identity proof
        authority-bearing object by default
        semantic event label
        replacement for Distinction-State

## Evidence

    Recorded structural support.

    Evidence may support distinction, relation, proof, governance, projection, or retention.

    Evidence is not proof unless tested under declared proof conditions.

## Execution Trace

    An ordered behavioral record.

    May include:

        task
        decision point
        action/tool call
        result
        failure
        repair
        retry
        test result
        projection
        proof request
        governance action

    A trace is not behavior proof.

    A trace is a realized trajectory through decision space.

## Exploitative Mode

    An admissibility mode used when the system prefers known stable paths.

    Allows bounded reuse of proven or retained structures.

    Blocks high-risk branch expansion and speculative mutation.

## Exploratory Mode

    An admissibility mode used when the system is discovering structure.

    Allows local exploration and uncertainty-bearing traces.

    Blocks mutation, identity commitment, training export, and agent authority expansion.

## Expose

    A T2 phase operator.

    Expose makes a structure, state, reference, or region available to an observer, process, or downstream operator without changing its identity or authority.

    Expose does not select, validate, interpret, authorize, or mutate.

## Exposure Authority

An authority-vector dimension governing what may be shown to users, agents, tools, interfaces, external systems, or training exports.

Projection authority does not imply identity authority.

# F

## Failure

    A structurally meaningful outcome where a candidate, transition, proof, projection, or authority request cannot proceed as requested.

    Failure is not garbage.

    Failure is structural evidence.

## Failure Legibility

    The invariant requiring failures to remain typed, retained, and routable.

    Failure types may include:

        reject
        defer
        quarantine
        underresolved
        unstable
        over-budget
        lineage-broken
        uncertainty-too-high
        authority-insufficient
        reconstruction-failed

## Failure Surface

    A decision-topology region where transitions repeatedly produce failure, instability, or constraint violation.

    Authority ceiling:

    mapped failure region

## Feature

    A chart-local pattern detected within a chart or structural signature.

    Examples:

        spectral peak
        energy spike
        token motif
        structural discontinuity
        tool retry cluster
        test failure recurrence

    Feature is not distinction.

    Authority ceiling:

        chart-local pattern

## Feature Extraction

    The structural-layer process that detects chart-local patterns.

    Feature extraction prepares inputs for distinction formation.

    It does not compare features across charts, resolve invariance, or form distinctions.

## Field

    A bounded space of possible states under constraint.

    Formal:

        F = { S | Θ }

    A field may be continuous, discrete, symbolic, energetic, probabilistic, relational, or computational.

## Field Differential

    The spatial and relational change of state quantities across local manifold regions.

    Field differentials describe:

        pressure
        gradient
        flow
        curvature
        stability
        collapse

    They upgrade point state vectors into local field behavior.

## FieldProfile

    A local field differential profile over an active Distinction-State.

    May include:

        salience_gradient
        activation_flux
        uncertainty_pressure
        constraint_pressure
        phase_gradient
        bias_gradient
        recurrence_flow
        retention_pressure
        persistence_stiffness
        curvature
        recursion_gradient

    Authority ceiling:

        field behavior evidence

## Flow

    Directional movement or transfer of activation, attention, uncertainty, recurrence, or influence across local runtime neighborhoods.

    Flow is field behavior.

    Flow is not proof.

## Four Substrate Contract

    The root architectural separation between:

        Native Emission / Payload
        Ledger Hypergraph
        Runtime Manifold
        Projection / Index

    These substrates must not collapse into one another.

## Full Proof

    A proof standard that tests all six invariant axes:

        composition
        magnitude
        alignment
        persistence
        separation
        convergence

    Required for identity commitment, recovery proof, reduction proof, merge/separation proof, governance proof, and agent-authorized action.

# G

## Global Scope

    A scope that applies system-wide.

    DME V2 forbids global scope by default.

    All authority, admissibility, proof, projection, and runtime behavior must declare scope.

## Governance

    The procedure that evaluates, grants, limits, configures, revokes, records, and audits authority and admissibility effects.

    Governance decides consequence.

    Proof does not.

    Admissibility does not.

    Runtime does not.

        mutation_candidate

        quarantine_candidate
        revocation_candidate

        bridge_candidate

        proposal only

        accept

        reject

        scoped consequence decision

## Governance-Ready Mode

        proof refs

        risk model

        monitoring plan

        recursion gradient

# H

## Hidden Loss

    A failure mode where structure is dropped, compressed, projected, filtered, or transformed without declaration.

    Hidden loss violates No Silent Loss and No Free Compression.

    A bounded compressed field representation used when full field geometry is too expensive or unstable.

        preserved axes
        omitted axes
        loss profile
        uncertainty
        reconstruction bounds

    authority ceiling

        A field hologram is not the field.

## Human Review

    A governance role.

    Human review is not magical authority.

        decision
        scope
        review basis
        authority vector change
        admissibility effect if any
        uncertainty posture
        reversibility/monitoring plan

# I

## Identity

    Structure that preserves invariance across declared transformations, perturbations, reconstruction, or context shifts.

    Identity is not:

        label
        meaning
        semantic claim
        recurrence alone
        projection
        salience
        authority

    Identity must be proven within scope.

## Inference

    A bounded derivation from structural evidence.

    In DME V2, inference must declare lineage, basis, uncertainty, authority ceiling, and non-claims.

    Inference is not authority.

## Information

    Distinguishable structure relative to observer, lens, and constraint.

    Information is not raw data.

    A decision topology edge connecting one DecisionNode or Distinction-State to another through an admissible realized or candidate transition.

    Statuses:

        realized
        rejected
        counterfactual
        failed
        repaired
        deferred
        quarantined

    Authority ceiling:

        transition candidate / trace

# U

## Uncertainty

    Unresolved distinguishability under declared observer, lens, and constraint.

    Uncertainty may arise from:

        insufficient resolution
        cross-basis disagreement
        missing axes
        unstable state evolution
        ambiguous boundaries
        lens bias
        incomplete provenance

    Uncertainty must be preserved, not silently collapsed.

Uncertainty Bound

    An admissibility axis determining whether unresolved distinguishability is acceptable for the current mode.

    High uncertainty may be admissible in exploratory or diagnostic mode.

    High uncertainty usually blocks governance mutation.

Uncertainty Entropy

    A state-vector quantity measuring unresolved distinguishability.

    Typical symbol:

        H

    May include:

        cross-basis disagreement
        missing-axis penalty
        instability penalty
        ambiguous boundary condition

## Uncertainty Preservation

    The law that uncertainty remains attached to the structure it qualifies until resolved, bounded, or explicitly deferred.

# V

## Validation

    The process of testing candidate survival under declared proof conditions.

    Validation does not automatically create authority.

## View

    A bounded projection of internal state.

    View is not source.

    Projection is a view.

# W

    ## Window

    A bounded source-local segment used by Structural Admission.

    Examples:

        audio time window
        token span
        JSON subtree
        execution trace slice

    Windowing does not create events, distinctions, proof, or authority by itself.

# X

## Cross-Axis Coupling

    A proof or runtime condition where change in one invariant axis affects another.

    Example:

        magnitude perturbation → separation drift
        alignment perturbation → convergence loss

    Cross-axis coupling can expose hidden pseudo-invariance.

# Z

## Zero-Authority Runtime

    The core runtime posture that active dynamics may reveal, propose, map, and route structure but cannot authorize consequence.

    Equivalent boundary:

    field dynamics may propose structure,
    but may not authorize structure.
