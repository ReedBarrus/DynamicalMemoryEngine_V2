# DME V2 — Runtime Map

## Purpose

The Runtime Map defines how structure moves through DME V2.

It is the implementation-facing movement map for the Architecture.

Architecture defines:

        > what DME V2 is.

Runtime Map defines:

        > how DME V2 runs.

DME V2 runtime is:

        > a distinction-first, admissibility-bound, proof-governed runtime sequence for converting native emissions into bounded structural distinctions, active runtime states, relation/topology candidates, proof results, governance decisions, and controlled projection/retention/agency feedback.

---

## Runtime Thesis

DME V2 does not begin with meaning.

DME V2 does not begin with events.

DME V2 begins with source-native emissions and derives structure through lawful transformations.

The runtime sequence is:

        native emission
        → structural exposure
        → chart/signature/feature formation
        → cross-basis distinction formation
        → active Distinction-State
        → field dynamics
        → admissibility evaluation
        → relation/topology candidates
        → decision topology where behavioral
        → proof execution
        → governance decision
        → projection / retention / agency feedback
        → recursion

Compressed:

        emission
        → structure
        → distinction
        → active state
        → relation/topology
        → admissibility
        → proof
        → governance
        → feedback

## Runtime Boundary Law

The strongest runtime boundary is:

        possible ≠ admissible
        admissible ≠ proven
        proven ≠ authorized
        authorized ≠ permanent

Expanded:

        source ≠ structure
        structure ≠ distinction
        distinction ≠ identity
        activity ≠ authority
        salience ≠ proof
        mapping ≠ mutation
        projection ≠ source
        retention ≠ identity
        agency ≠ sovereignty

Every runtime phase must preserve this boundary.

### 5.1 Attention-Energy

Attention-Energy determines which active Distinction-States are processed, perturbed, related, stabilized, or allowed to decay.

It measures:

        activation
        salience
        influence
        uncertainty pressure
        processing priority
        field deformation

It is not authority.

Runtime Budget Law
        Σ attention_weight ≤ global_budget
        Σ activation_energy ≤ activation_budget

No Distinction-State may accumulate unlimited influence.

### 5.2 Field Differential Dynamics

Field Differential Dynamics describes pressure, gradient, flow, curvature, recurrence, and persistence behavior across local manifold regions.

FieldProfile may include:

        salience_gradient
        activation_flux
        uncertainty_pressure
        constraint_pressure
        phase_gradient
        bias_gradient
        recurrence_flow
        retention_pressure
        persistence_stiffness
        curvature
        recursion_gradient

Field differentials are local.

They are not proof.

### 5.3 Interference Process

Interference is local, bounded perturbation between active Distinction-States.

It produces:

        state deltas
        attention shifts
        uncertainty shifts
        curvature deltas
        influence values
        interaction traces

Output:

        InterferenceTrace

Interference is not relation by itself.

A single perturbation does not establish stable dependency.

### 5.4 Relation Candidate Formation

A RelationCandidate forms when interference effects become repeatable, measurable, and context-valid.

Relation types may include:

        similarity
        alignment
        recurrence
        separation
        transition
        convergence

A relation is stabilized dependency, not raw similarity.

### 5.5 Topology Candidate Formation

A TopologyCandidate forms when relation candidates stabilize into persistent relational form.

Examples:

        neighborhood
        trajectory
        basin
        attractor
        recurrence field

Topology is not semantic clustering.

It is persistent relational structure under repeated interference.

Runtime Manifold Constraints

The Runtime Manifold may not:

        create authority
        write ledger truth directly
        turn salience into identity
        treat curvature as proof
        treat basin formation as governance
        make recurrence sovereign
        perform unbounded all-to-all interaction

Runtime Field Rule
        field dynamics may propose structure,
        but may not authorize structure.

### 7.2 Decision Topology Mapping

Decision Topology Mapping applies when runtime behavior or execution traces contain branching structure.

        A DecisionNode is detected when:

                |T_admissible(Dᵢ)| ≥ 2

                AND H(Dᵢ) ≥ H_min
                AND realized_transition exists or is being selected
