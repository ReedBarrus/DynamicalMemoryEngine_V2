## Current posture

- Door One export targets are limited to inspection, replay, and narrow consult surfaces that keep provenance intact
- external field inputs are limited to declared packets with provenance support and Door One workflow coverage
- no SupportRegime or SymbolicRegime activation is implied—this note routes interest, it does not authorize delivery

## Deferred items

- broader program-level roadmaps remain in the Memory archive until V1 review or an explicit migration packet brings them forward
