# Resonance Vocabulary Translation Table

## Status

**Status:** Draft translation reference
**Role:** Boundary-support document
**Placement intent:** Door Two README subfolder
**Purpose:** Translate high-load Resonance primitives into DME-safe language so field-mechanism insight can be used without importing unbounded ontology, authority, or settlement claims.

---

## Purpose

This document provides a **translation table** for Resonance vocabulary as it crosses into DME.

It is not a rejection of Resonance terminology. It is a **containment and precision tool**.

The problem it solves is simple:

> Resonance primitives are compact, powerful, and often carry ontology, validity, and settlement implications in a single term. DME cannot safely import those terms without narrowing them.

This document therefore defines, for each important Resonance primitive:

1. what the term means inside Resonance,
2. the nearest DME-safe translation,
3. what is preserved in translation,
4. what must be explicitly excluded,
5. how the term may be used inside DME artifacts.

---

## Use Rule

When a Resonance term appears inside DME-side writing, review, or mechanization planning, prefer:

* the **DME-safe translation**, or
* the original term followed immediately by its DME-safe translation.

Example:

* “phase lock (alignment stability under declared lens)”
* “cavity mode fit (boundary-fit stability under declared conditions)”

This keeps the conceptual bridge intact while preventing claim inflation.

---

## Translation Posture

All translations in this document obey the following rule:

> DME may preserve mechanism-language from Resonance while refusing any silent import of ontology, truth, settlement, canon, or self-authorizing legitimacy.

If a translated use violates that rule, it is out of bounds.

## Detailed Term Translations

## 1. Spectral State

### Resonance-side meaning

A spectral state is treated as the primary representational unit of the system: the frequencies, amplitudes, and phases that compose the underlying reality of the signal or network state.

### DME-safe translation

**Privileged structural representation**
or
**Spectral support representation for bounded families**

### What is preserved

* compact structural encoding,
* harmonic composition,
* pattern-level comparability,
* useful support for retrieval, comparison, reconstruction, and deviation review.

### What is excluded

* the claim that the spectral state is necessarily the final or native reality,
* the claim that a spectral representation exhausts the object,
* the claim that structural compactness implies ontological priority.

### Allowed DME usage

* “The candidate is represented in a privileged spectral form under the declared lens.”
* “This replay family admits a stable spectral support representation.”

### Disallowed DME usage

* “The spectral state is the object itself.”
* “The spectral state proves the ontology.”

## Translation Rules for Writing

### Preferred pattern

When using Resonance language inside DME docs, prefer this format:

**Original term (DME-safe translation)**

Examples:

* coherence (structural support)
* phase lock (alignment stability under declared lens)
* attractor basin (stable support neighborhood)
* cavity mode fit (boundary-fit stability under declared conditions)

### When to drop the original term

Once the context is fully within DME review/governance language, use only the DME-safe translation.

Example:

* Prefer “boundary-fit stability” over repeated “cavity mode fit” inside candidate packet review if the Resonance framing is no longer needed.

### When to preserve the original term

Preserve the original term when:

* comparing directly against Resonance materials,
* teaching the boundary,
* explaining CPD fields,
* documenting translation history.

## Translation Heuristic

Whenever a Resonance primitive appears, ask:

1. Is this term naming a **mechanism**, a **representation**, an **interpretation**, an **identity claim**, or a **settlement claim**?
2. Which parts of that meaning are useful in DME?
3. Which parts would over-import ontology or authority?
4. What narrower DME-safe phrase preserves the useful part?

This heuristic should guide future additions to the table.
