## 2. Why this experiment exists

Door One is now materially stabilized at the runtime, provenance, inspection, preservation, and anti-bypass seam levels.

That means the next honest question is no longer only:

**Can Door One run?**

The next honest question is:

**Can Door One answer at least one real diagnostic question from a real source with less ambiguity than a naive inspection baseline?**

This experiment exists to answer that question before:

- transform-lens changes,
- passive multi-scale ingest work,
- or Door Two design pressure.

It should be run once on the current Door One baseline, then repeated after later structural-lens changes such as:

- real FFT evaluation,
- wavelet structural lens evaluation,
- passive multi-scale ingest extension.

## 3. Experiment objective

Demonstrate that Door One can:

1. preserve lawful provenance from a real device source,
2. detect a structural difference across at least one controlled regime change,
3. show whether recurrence after perturbation is true return or only superficial similarity,
4. present the result through lawful inspection surfaces without semantic inflation.

This is the minimum usefulness threshold for Door One as an instrument.

## 5. Minimal regime structure

The experiment should contain at least three bounded conditions:

### Phase 1 — Baseline

Record a stable repeated pattern.

Purpose:
- establish recurrence under one regime,
- establish provenance and lawful ingest from a real source.

### Phase 2 — Perturbation

Introduce one deliberate structural change.

Examples:
- change frequency or rhythm,
- add transient noise,
- switch input condition,
- alter sensor behavior in one bounded way.

Purpose:
- test whether Door One detects structural drift or novelty.

### Phase 3 — Return or alternate stable regime

Either:
- return to the original pattern,
- or settle into a different repeated pattern.

Purpose:
- test whether Door One distinguishes:
  - true return,
  - partial resemblance,
  - and genuinely different recurrence.

- canon,

### 10A. Current structural-lens comparison result

The current real-device baseline has now been compared against three later structural-lens variants:

- real FFT cleanup
- wavelet structural lens
- passive declared multi-scale on the FFT/Hann baseline

#### Real FFT cleanup

Current finding:

- real FFT cleanup preserves the same normalized band-profile geometry as the current DFT-based baseline under the present comparison basis,
- replay separability, return-like convergence, transition support, and concentration support remain unchanged,
- the practical gain is speed and later receipt honesty, not new structural discrimination.

Working meaning:

real FFT remains a valid future backend/runtime optimization candidate,
but not a new analytical lens for the present real-source families.

#### Wavelet structural lens

Current finding:

- Haar and D4 wavelet paths preserve the gross replay story only partially,
- both degrade the narrower-band `400 Hz` family relative to FFT/Hann,
- the narrow-band concentration distinction is weakened or destroyed,
- neither wavelet path improves boundary or return behavior enough to justify replacing the baseline.

Working meaning:

for the current steady-state real-source families, FFT/Hann remains the better active structural lens.

Wavelets may be revisited later for transient-dominant cohorts.

#### Passive multi-scale on FFT/Hann baseline

Current finding:

- short, medium, and long declared lenses all preserve the core replay story,
- short remains noisier and more fragmentation-prone,
- long yields only marginal steady-state separability gain while introducing real temporal smearing risk near phase boundaries,
- medium remains the best balance point for the current real-source families.

Working meaning:

the current medium FFT/Hann posture is not arbitrary.
It remains the preferred active real-source lens for the present experiment families.

#### Summary

For the current phase:

- keep medium FFT/Hann as the active real-source baseline,
- treat real FFT as a later backend optimization candidate,
- treat wavelets as deferred for current steady-state families,
- treat passive short/long scales as lawful comparison lenses with known caveats, not as active defaults.

## 14. Practical lesson from the first audio capstone

The first audio-file capstone on `220-440hzPulse.wav` established an important lesson for real-device work:

**temporal support matters.**

The baseline finding was not merely that Door One could distinguish baseline / perturbation / return.

The stronger finding was that a structurally real perturbation may fail to stabilize as runtime structure if the declared temporal support lens is too coarse relative to the event duration.

### Source and declared ingest lens

Source:
- file-backed WAV source
- `source_id = door1.audio.220_440hz_pulse.v1`
- `channel = mono_master`
- `modality = audio_amplitude`

Declared ingest lens:
- original sample rate: 48000 Hz
- ingested sample rate: 2400 Hz
- declared decimation factor: 20

### Coarse temporal support case

Initial slice runner window family:
- `base_window_N = 4096`
- `hop_N = 2048`

At 2400 Hz, this corresponds to a window duration of approximately 1.7067 s.

The perturbation slices were each only 1.25 s long.

Observed result:
- baseline slices stabilized cleanly,
- return slices stabilized cleanly,
- perturbation slices were distinguishable at cross-run comparison,
- but perturbation slices collapsed to zero structured state output.

Interpretation:
the perturbation was not absent.
It was under-supported by the declared temporal analysis window.

### Tighter temporal support case

Adjusted slice runner window family:
- `base_window_N = 1024`
- `hop_N = 512`

At 2400 Hz, this corresponds to a window duration of approximately 0.4267 s.

Observed result:
- baseline slices remained stable and reproducible,
- perturbation slices became representable as structured runs,
- at least one perturbation slice expressed multiple basins / segments / transitions,
- return slices remained stable and reproducible,
- return remained closer to baseline than the stronger perturbation slice.

### Current implication for real-device work

Before changing transform family or adding multi-scale ingest, Door One should treat temporal window policy as a first-class practical lens variable for real-device diagnostics.

For short modulation / transition events in roughly the 200–500 ms range, tighter window families may be necessary to detect structure honestly.

This does **not** authorize hidden scale selection or prediction.

It supports declared temporal-lens adjustment within Door One’s lawful ingest / structural scope only.

## Real-source replay refinement

The first file-backed microphone cohort showed a lawful and informative divergence from the synthetic replay line.

The cohort did not exhibit `oscillatory_exchange`; all phases remained `one_way_drift`.

However, perturbation and return remained strongly legible in band-profile geometry, and return was much closer to baseline than perturbation.

Working implication:

for real-source microphone replay, the first useful recovery diagnostic may be band-profile return geometry rather than exchange-regime recovery.

This remains below canon and does not authorize hidden lens changes or semantic interpretation.
