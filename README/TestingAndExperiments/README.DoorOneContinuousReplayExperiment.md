## 2. Why this experiment exists

The first audio capstone already showed that Door One can preserve lawful provenance and distinguish stable baseline, active perturbation, settling modulation, and stable return when the declared temporal support lens is appropriate.

That was enough to show that Door One can become structurally discriminative.

The next honest question is stronger:

**Can Door One preserve replay-honest structural distinction across a bounded continuous real-source loop while staying clear about what was retained, what was pruned, what can still be replayed, and where ambiguity begins?**

This experiment exists to answer that question before:

- passive multi-scale ingest work,
- transform-family changes,
- or later-layer semantic pressure.

---

## 3. Experiment objective

Demonstrate that Door One can:

1. preserve lawful provenance from a simple repeatable real source,
2. detect structural differences across repeated bounded regime changes,
3. distinguish true return from partial resemblance or alternate stable recurrence,
4. preserve enough retention support for honest replay after live pruning,
5. expose the result through lawful inspection surfaces without semantic inflation.

This is a stronger threshold than the first file-backed audio capstone.  
It tests not only structural sensitivity, but replay-honest retention.

## 4. Recommended source

Use one real numeric source that is easiest to repeat honestly.

Preferred options:

### A. Sampler-backed audio / microphone source

Examples:
- repeated tapping pattern,
- repeated tone pattern,
- sustained hum with one controlled disturbance,
- repeated pulse from a speaker or phone tone.

### B. Analog sampler-backed signal

Examples:
- repeated mechanical vibration,
- simple periodic voltage-like source,
- simple sensor movement with one bounded perturbation.

### C. Existing sampler-backed device path

If the current analog sampler path is easiest to access, it is acceptable as the first continuous replay loop source, provided the source is genuinely non-synthetic.

### Source selection rule

Choose whichever source is:

- easiest to repeat honestly,
- easiest to reset,
- easiest to perturb in one controlled way,
- easiest to preserve provenance for,
- and simple enough that ambiguity is diagnostic rather than source-chaos.

Do **not** optimize for complexity.  
Optimize for repeatability.

## 7. Session structure

Run one bounded session composed of repeated small runs rather than one giant recording.

### Phase A — Baseline

Preferred minimum:
- 3 runs

Purpose:
- establish stable recurrence,
- establish repeatability,
- establish the control regime.

### Phase B — Perturbation

Preferred minimum:
- 3 runs

Introduce exactly one perturbation axis.

Examples:
- cadence change,
- amplitude change,
- one inserted transient,
- altered interval spacing,
- changed sensor posture.

Purpose:
- test detectable structural drift or novelty.

### Phase C — Return

Preferred minimum:
- 3 runs

Return to baseline setup.

Purpose:
- test whether recurrence is a true return,
- or only partial resemblance.

### Optional Phase D — Alternate stable regime

Preferred minimum:
- 2 runs

Use a different but still stable repeated pattern.

Purpose:
- force the system to distinguish:
  - baseline return,
  - alternate stable recurrence,
  - and simple novelty.

---

## 8. Perturbation rule

Use only **one perturbation axis at a time** for the first loop.

Do not combine:

- cadence change,
- amplitude change,
- noise injection,
- source-position shift,
- and sensor-state shift

in the same first experiment.

If the goal is honest comparison, one change per experiment is the correct rule.

## 13. Failure classes

### F1 — Source failure

The source itself is not repeatable enough.

Examples:
- unstable baseline,
- inconsistent perturbation,
- return phase not actually reset,
- uncontrolled environmental contamination.

### F2 — Ingest failure

The source may be fine, but ingest posture is not.

Examples:
- timestamp ambiguity,
- source/stream identity ambiguity,
- bad flush behavior,
- missing lawful ingest fields.

### F3 — Structural under-resolution

The source and ingest are lawful, but the lens is too coarse or poorly matched to the event horizon.

### F4 — Runtime discrimination failure

Door One runs lawfully but fails to separate baseline / perturbation / return in its structural outputs.

### F5 — Replay failure

The system once saw the distinction, but retained tiers no longer support the replay question honestly.

### F6 — Inspection failure

The evidence exists, but the inspection path muddies or overstates the result.

### F7 — Retention failure

The selected retention tier was too aggressive for the declared replay question.

Examples:
- digest survives but receipt lineage is needed,
- pinned packet is missing key receipt refs,
- pruning destroys the actual experiment question.

### Perturbation families tested

- amplitude

### Current result

All four tested perturbation families showed the same replay pattern:

- baseline: `oscillatory_exchange`
- perturbation: weakened or collapsed exchange regime
- return: `oscillatory_exchange`

Family-level classifications:

- amplitude → `exchange_recovers_on_return`
- boundary_detuning → `exchange_recovers_on_return`
- harmonic_offset → `exchange_recovers_on_return`
- noise_depth → `exchange_recovers_on_return`

Cross-family summary:

- `resilience_surface_class = broad_recovery_pattern`

### Working meaning

Within the current synthetic replay slice, oscillatory exchange appears robust across multiple independent disruption axes.

This strengthens the current Door One reading:

the exchange condition is not only support-horizon-sensitive and boundary-conditioned;
it also shows bounded replay resilience under multiple isolated perturbations when the perturbation is removed.

This remains a structural replay observation only.

It is not yet:

- canon,
- ontology,
- trusted prediction,
- or a general law across all source families.

### Measurement discipline note

One proposed harmonic-offset family was rejected during probe construction because it triggered the current `oscillatory_exchange` classifier through a chaotic sign-flip path rather than through the coherent period-2 mechanism under investigation.

The accepted harmonic-offset family preserves the nominal measurement lens while moving the harmonic off the active band edge, allowing cross-family comparison without changing the observational frame.

This preserves probe honesty.
