Dynamical Memory Engine — Glossary
Status

This document is a supporting terminology and definition note for the Dynamical Memory Engine (DME).

It is not constitutional authority.

It does not override:

README.MasterConstitution.md
README.WorkflowContract.md
README.CrossFieldFormalSchema.md

Its purpose is narrower:

define current DME terminology cleanly,
align local project language with current constitutional meaning,
reduce naming drift,
preserve field-facing clarity,
distinguish DME-local terms from reused field terms,
and keep active Door One language from silently inflating into ontology, truth, or canon.

This glossary should be read together with the Cross-Field Formal Schema. It is a term-definition surface, not a proof surface.

1. Constitutional posture

DME operates under a strict layer order:

measurement → structure → runtime memory → recognition
→ canon → prediction → agency → ecology → symbolic → meta

For Door One:

runtime is not canon,
query is not truth,
substrate is not ontology,
replay is lens-bound,
read-side interpretation remains below promotion.

Accordingly, glossary definitions must not silently turn:

structure into meaning,
recurrence into truth,
substrate organization into ontological attractor claims,
or read-side usefulness into promotion authority.
2. How to read this glossary

Each entry uses the following fields where useful:

Definition — what the term means in DME
Layer / seam — where it belongs
Authority posture — what kind of thing it is allowed to be
Nearest field language — the closest standard terminology when helpful
Must not be confused with — common drift risks
Notes — bounded usage rule

Not every entry needs every field, but the structure should stay recognizable.

3. Core layer and authority terms

Definition
The lowest active lawful layer. It captures what was observed, from where, and under what timing/provenance conditions.

Layer / seam
Signal Space

Authority posture

Nearest field language

Must not be confused with
structure, interpretation, canon

9. Cross-field translation reminders
State

Use carefully. In many fields this implies full system configuration or formal state-space status. In Door One, prefer more local terms like:

runtime artifact
structural profile
trajectory position
segment status
unless formal state-space language is truly intended.
Attractor

Use conservatively. In active Door One, prefer:

structural neighborhood
recurring region
basin-like organization
unless you are explicitly making a bounded analogy rather than a proven claim.
Coherence

This term may be used only with explicit support and declared layer. It should not become a free-floating approval word.

Prefer:

recurrence support
structural stability
concentration
persistence
unless a more specific coherence definition is declared.
Memory

Always qualify where possible:

runtime memory
substrate memory
canonical memory
interpretive memory

Do not use unqualified “memory layer” across unlike meanings.

10. Terms intentionally avoided or fenced

Avoid or heavily qualify:

ontology when only structure is meant
truth when only recognition or replay success is meant
canonical when only recurrent or stable is meant
understanding when only comparison or interpretation is meant
dimension when only feature or response axis is meant
intelligence stack when Door One alone is intended

These fences preserve legitimacy and cross-field honesty.

11. Working summary

This glossary exists to keep DME language precise enough that:

active Door One work stays lawful,
cross-field speech stays accountable,
local project terms remain legible,
and useful analogies do not silently harden into equivalence or truth claims.

It should be updated when term meaning changes materially, not merely when a new experiment is added.

DME is a signal-driven structural memory engine that acquires discrete-time signals, transforms bounded windows into structural feature representations, organizes those representations into trajectories and recurring structural neighborhoods under declared lenses, and uses replay-honest read-side probes to characterize recurrence, transition, and return-like convergence while keeping trusted canon formation outside normal runtime operation
