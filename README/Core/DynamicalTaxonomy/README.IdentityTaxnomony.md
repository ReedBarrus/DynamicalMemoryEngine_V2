## 1. Why this note exists

DME now has enough structure, replay, retention, observability, feedback, memory, workflow, and continuity pressure that the project needs one explicit place to answer a recurring question:

**What kind of identity is being preserved here, and at what class of scope does that preservation remain lawful?**

Without a bounded identity hierarchy, several distortion risks become active:

- mechanical continuity being mistaken for symbolic continuity,
- symbolic continuity being mistaken for human continuity,
- usefulness being mistaken for sameness,
- memory support being mistaken for preserved identity,
- observability or feedback richness being mistaken for identity depth,
- and lower-layer continuity language being silently stretched upward into broader identity classes.

This note exists to prevent those drifts.

---

## 2. Core rule

**In DME, identity is not treated as one flat universal object. Identity is treated as a hierarchy of bounded continuity classes, where each class answers a distinct bounded question under a distinct support, boundary, memory, observability, and constraint regime.**

Corollary rules:

- resemblance is not automatically identity,
- preserved memory is not automatically preserved identity,
- lower-class continuity is not automatically higher-class continuity,
- symbolic continuity is not automatically human continuity,
- and repeated usefulness is not automatically identity conservation.

If the active bounded question changes, the support basis weakens, the active constraints drift, or a broader class is being claimed than the current support actually justifies, the system must not silently preserve the stronger identity claim.

It must instead narrow, downgrade, defer, split, reclassify, or emit unresolved posture.

## 4. Definition of identity

In DME, **identity** means:

> the continued lawful ability to treat a structure, object, process, packet, replay surface, retained surface, symbolic object, or human-modeled continuity object as the same bounded object of inquiry because the same bounded question remains answerable under the same declared identity-class constraints with sufficient surviving support and without authority inflation.

This means identity is always:

- question-relative,
- class-relative,
- constraint-relative,
- support-relative,
- memory-relative,
- boundary-relative,
- and mechanization-relative where a real path matters.

Identity is **not** guaranteed merely because:

- a name remains the same,
- a shape remains similar,
- a summary feels coherent,
- memory remains in some weaker class,
- a visible surface appears stable,
- a structure is still useful,
- or a behavior repeats.

## 5. Identity is not constraint, boundary, observability, feedback, or memory

Identity depends on these, but is not reducible to any one of them.

### Constraint
Constraint governs:
- what is lawfully allowed,
- what must remain preserved,
- what may still count as valid.

### Boundary
Boundary governs:
- what is inside vs outside,
- what may cross,
- what may remain bounded.

### Observability
Observability governs:
- what can become detectable, registered, and state-consequential.

### Feedback
Feedback governs:
- how interaction becomes future-state-shaping.

### Memory
Memory governs:
- what remains preserved and reusable across time.

### Identity
Identity governs:
- what bounded object remains the same under the active class of continuity claim.

Therefore:

- identity requires support from other classes,
- but no single parent class is sufficient by itself,
- and stronger identity classes require a richer and more coupled support stack than weaker ones.

One-line distinction:

**Constraints define lawfulness. Boundaries localize it. Observability and feedback register and propagate it. Memory carries what survives it. Identity is the bounded sameness that remains lawful through all of them.**

## 6. The five identity classes

The current bounded taxonomy recognizes five primary identity classes.

---

### 6.1 I0 — Mathematical identity

#### Definition

Identity under a formal system where sameness is determined by exact equivalence, valid substitution, or rule-governed invariance.

#### Bounded question

**When is this object exactly the same under the active formal system?**

#### Primary role

- govern exactness,
- govern equivalence and substitution,
- provide the narrowest and cleanest identity class.

#### Dominant supports

- formal constraints,
- rule-valid transformation,
- exact symbolic preservation.

#### Common examples

- equality,
- formal equivalence class,
- exact symbolic identity under declared rules.

#### Main failure modes

- false equivalence,
- invalid substitution,
- category error,
- rule drift.

#### One-line summary

**I0 is exact formal sameness.**

---

### 6.2 I1 — Geometric identity

#### Definition

Identity under lawful transformation where a form, topology, or structural relation remains the same strongly enough to count as the same object of geometric inquiry.

#### Bounded question

**When does this form remain the same under allowed transformation?**

#### Primary role

- govern form-preserving change,
- preserve topology or invariant relation sets,
- extend identity beyond exact formal equivalence into lawful transform-space continuity.

#### Dominant supports

- formal constraints,
- structural constraints,
- transform boundaries,
- invariant-preserving remaps.

#### Common examples

- same topological object under allowed deformation,
- same shape class under declared invariant set,
- same relational form under bounded geometric transformation.

#### Main failure modes

- rupture,
- invariant loss,
- topology break,
- transform equivalence overclaim.

#### One-line summary

**I1 is lawful form-preserving sameness.**

---

### 6.3 I2 — Mechanical / topological identity

#### Definition

Identity of a bounded structure that remains the same under perturbation, replay, retention, transformation, and lawful interaction strongly enough to support the same bounded question under non-symbolic DME conditions.

#### Bounded question

**When does this bounded structure remain the same object under perturbation, retention, feedback, and support-preserving transformation?**

#### Primary role

- serve as the active DME identity kernel class,
- govern bounded continuity for replay, retention, workflow, and runtime surfaces,
- provide the main lawful identity class for current Door One work.

#### Dominant supports

- structural constraints,
- boundary constraints,
- dynamical constraints,
- persistence constraints,
- coupling/feedback constraints,
- cross-layer consistency constraints,
- retention and identity boundaries,
- M1–M3 memory classes,
- O1–O3 and F1–F3 observability/feedback classes where applicable.

#### Common examples

- replay-support-bounded structural object,
- retained continuity object,
- workflow object preserving the same bounded seam,
- same family/local structure under bounded perturbation and declared lens.

#### Main failure modes

- question drift,
- constraint drift,
- support collapse,
- false continuity through similarity,
- mechanization gap,
- cross-layer fragmentation,
- distortion beyond identity threshold.

#### One-line summary

**I2 is bounded structural sameness under perturbation and support pressure.**

---

### 6.4 I3 — Symbolic identity

#### Definition

Identity of a symbol-bearing structure or agent-like continuity object that remains the same across changing roles, meanings, abstractions, or narratives strongly enough to support the same bounded symbolic question.

#### Bounded question

**When does this symbol-bearing object remain the same across changing context, role, or meaning-bearing transformation?**

#### Primary role

- extend identity into symbolic and interpretive regimes,
- provide a place for narrative, role, and symbolic continuity to be discussed without collapsing them into I2.

#### Dominant supports

- formal constraints,
- dynamical constraints,
- persistence constraints,
- coupling constraints,
- cross-layer consistency constraints,
- symbolic memory and role continuity,
- stronger interpretive observability and feedback,
- symbolic/contextual boundaries.

#### Common examples

- role continuity under bounded symbolic change,
- same symbolic object under lawful narrative extension,
- meaning-bearing continuity object.

#### Main failure modes

- semantic drift,
- narrative fracture,
- role collapse,
- symbolic overcompression,
- symbolic continuity overclaim from only mechanical support.

#### One-line summary

**I3 is bounded symbolic sameness under meaning-bearing transformation.**

---

### 6.5 I4 — Human earthly identity

#### Definition

Identity of a human person modeled as a bounded continuity object across embodied, behavioral, autobiographical, relational, and symbolic change under earthly cognition and lived causal continuity.

#### Bounded question

**When does a human remain the same person across bodily, cognitive, symbolic, emotional, behavioral, and social change?**

#### Primary role

- provide the current practical ceiling of the identity hierarchy,
- preserve a lawful place for person-level continuity without collapsing it into symbolic-only or mechanical-only identity.

#### Dominant supports

- structural constraints,
- dynamical constraints,
- persistence constraints,
- coupling constraints,
- cross-layer consistency constraints,
- autobiographical, behavioral, and relational memory,
- embodied and social observability / feedback,
- embodied, cognitive, behavioral, and relational boundaries,
- symbolic continuity where applicable.

#### Common examples

- human autobiographical continuity,
- behavioral and relational continuity through lived change,
- person-level identity under bounded earthly cognition.

#### Main failure modes

- fragmentation,
- dissociation,
- autobiographical rupture,
- symbolic/social identity split,
- embodied discontinuity,
- false person-level continuity inferred from only symbolic or mechanical support.

#### One-line summary

**I4 is bounded human person-level sameness under embodied and symbolic life continuity.**

## 7. Identity hierarchy rule

The five classes form a hierarchy, not a blur:

- **I0** = exact formal sameness
- **I1** = lawful form-preserving sameness
- **I2** = bounded structural sameness under perturbation/support
- **I3** = bounded symbolic sameness
- **I4** = bounded human earthly sameness

The hierarchy should remain explicit.

A higher identity class may inherit from lower classes where applicable, but it must not erase them silently.

Examples:

- I2 may depend on I1-like invariants without becoming only geometric identity,
- I3 may depend on I2 continuity without collapsing symbolic identity into mechanics,
- I4 may depend on I3 continuity without reducing human identity to narrative alone.

The system must not silently round lower-class continuity upward.

Examples:

- I2 is not automatically I3
- I3 is not automatically I4
- I1 similarity is not automatically I2 continuity
- I0 formal equivalence is not automatically person-level sameness

## 9. Identity functions

Each identity class can be analyzed through three functions.

### 9.1 Sameness function

**What does it mean, at this class, for the object to remain the same?**

Examples:
- exact equivalence,
- preserved topology,
- support-preserving structural continuity,
- symbolic continuity,
- embodied-relational continuity.

### 9.2 Support function

**What must survive strongly enough for that sameness claim to remain honest?**

Examples:
- formal validity,
- invariants,
- bounded question preservation,
- boundary integrity,
- memory support,
- feedback consequence,
- cross-layer coherence.

### 9.3 Failure function

**What break occurs when the class can no longer preserve its own bounded question honestly?**

Examples:
- false equivalence,
- rupture,
- degraded identity,
- semantic drift,
- fragmentation.

This three-function view prevents identity from collapsing into vague “self” language.

## 10. Identity is hierarchical, not universal-flat

Several surfaces can look identity-preserving while only preserving a weaker class.

### 10.1 Mechanical continuity is not automatically symbolic continuity

A system may preserve:
- bounded structure,
- replay support,
- memory support,
- and perturbation continuity,

while failing to preserve:
- role continuity,
- interpretive continuity,
- symbolic attractor continuity.

### 10.2 Symbolic continuity is not automatically human continuity

A symbolic object may preserve:
- narrative continuity,
- role continuity,
- meaning-bearing coherence,

while failing to preserve:
- embodied continuity,
- autobiographical continuity,
- behavioral continuity,
- relational continuity.

### 10.3 Reuse is not automatically identity

Useful reapplication may preserve:
- memory,
- support,
- function,

without preserving the same identity class.

### 10.4 Similarity is not automatically class preservation

Surfaces may remain:
- familiar,
- coherent,
- useful,
- recurrent,

while still failing the bounded identity question at the active class.

## 11. Identity-bearing vs support-only posture

A central dynamics distinction is:

### Identity-bearing

A structure supports a real bounded identity claim at its current class.

### Support-only

A structure remains useful as support, lineage, display, memory, symbolic trace, or behavioral trace, but does not justify the stronger identity claim.

### Residual continuity

A structure still appears related to the prior object, but no longer supports the prior identity class honestly.

This distinction should remain explicit.

The system must not casually call something identity-preserving when it is only:

- memory-bearing,
- support-bearing,
- similar,
- narratively convenient,
- replay-shaped,
- or residually related.

## 21. Identity failure modes

Identity most commonly fails through the following modes.

### 21.1 False equivalence

A weaker resemblance is treated as same-object continuity.

### 21.2 Question drift

The object is now answering a different bounded question while preserving the old identity language.

### 21.3 Constraint drift

The active class, lens, scope, or allowed-use conditions changed while preserving the stronger identity claim.

### 21.4 Support collapse

Not enough invariants, lineage, memory, or observable consequence survive for the class being claimed.

### 21.5 Mechanization gap

The identity claim depends on a real bounded path that does not actually exist.

### 21.6 Class inflation

A lower identity class is silently overread as a higher one.

### 21.7 Cross-layer fragmentation

Different surfaces remain separately coherent, but not as one identity object at the active class.

### 21.8 Distortion overrun

Representation, compression, or summary collapses distinctions strongly enough that the same bounded identity question can no longer be answered honestly.

## 23. Identity outcomes

A practical identity audit should produce one of the following broad outcomes.

### 23.1 Conserved identity
The same bounded question remains answerable at the active class under the same declared constraints with sufficient support and mechanized basis.

### 23.2 Conserved but narrowed identity
The same class remains lawful only under stricter use, scope, or support conditions.

### 23.3 Degraded identity
Useful continuity remains, but not enough to preserve the old identity class without qualification.

### 23.4 New identity class required
The current object now behaves as a different kind of bounded identity object and should be reclassified.

### 23.5 Support-only continuity
Related support remains, but no strong identity claim should be preserved.

### 23.6 Unresolved identity
Support is too weak to justify either strong continuity or strong break.

## 25. Decision rule

Preserve a stronger identity claim only when all of the following are true strongly enough for the active bounded question and class:

the identity class is explicit,
the same bounded question remains active,
the same declared constraints remain active,
enough support survives for that class to remain answerable honestly,
any required memory, boundary, observability, and feedback conditions for that class still hold strongly enough,
and any required implementation path is actually mechanized.

If not, the structure may still be useful.

But it must not silently preserve the stronger identity class.

## 26. Scope ceiling and expansion rule

The current practical hierarchy ceiling is:

human earthly identity

This ceiling is chosen because it is the highest range for which direct feedback, bounded modeling, and real continuity pressure remain practically discussable without widening into uncontrolled metaphysical scope.

Expansion rule:

new identity scopes may be added as new classes,
not by silently stretching lower classes upward,
and not by widening current classes until their bounded question blurs.

Revision rule:

within-scope refinement is lawful,
scope expansion requires a new class or explicit revision class,
and no broader class should be treated as active simply because it is conceptually attractive.

## 27. One-line practical rule

In DME, identity is a hierarchy of bounded continuity classes, and the same thing is only the same thing at a given class while the same bounded question remains supportable, constrained, and mechanized enough to be answered honestly at that class without inflation.
