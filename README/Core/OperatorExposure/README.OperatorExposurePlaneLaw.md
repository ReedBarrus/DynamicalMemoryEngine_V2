## 1. Why this note exists

DME now has enough runtime, replay, retention, visualization, packet, and semantic pressure that one specific problem has become unavoidable:

**the system cannot be trusted as an instrument if the operator cannot inspect what each operator seam is actually exposing.**

The recent surface work exposed several practical risks:

- structural output being replaced by labels, counts, or posture text,
- support/accounting surfaces being mistaken for first-order exposure,
- clean app composition being mistaken for truthful exposure,
- render planes becoming semantic "viewers" instead of structural instruments,
- and multi-layer surfaces causing raw structure, support, and interpretation to cohabit one seam.

DME already has a lawful sequence for development:

- define the seam,
- define what is real at that seam,
- define what remains below claim,
- preserve bounded closure,
- and then mechanize the next layer upward.  

This note applies that same discipline to operator-facing surfaces.

One-line summary:

**An operator exposure plane exists to expose what an operator seam actually produces, before support, semantics, or review are allowed to close over it.**

---

## 2. Core rule

**Every operator seam that produces first-order inspectable state should be able to expose its emitted output through a lawful operator exposure plane before support, semantic, or review layers are allowed to substitute for that output.**

Corollary rules:

- exposure comes before interpretation,
- primary operator output comes before support output,
- support output comes before semantic overlay,
- semantic overlay comes before review posture,
- and no later layer may stand in for a missing earlier layer.

If a surface cannot directly expose first-order operator output for the bounded question being asked, it must not be treated as a lawful operator exposure plane.

It must instead be classified as one of:

- support surface,
- derived inspection surface,
- semantic overlay surface,
- review surface,
- mixed surface,
- or unresolved surface requiring seam clarification.

One-line summary:

**An operator plane is lawful only when the operator can directly inspect what the seam produced before any stronger layer starts explaining it.**

## 4. Definition of an operator exposure plane

In DME, an **operator exposure plane** means:

> a bounded exposure surface that renders the first-order emitted output of a specific operator seam directly in the seam's own representational terms before support, semantic, or review layers are introduced.
This means an operator exposure plane is always:

- seam-relative,
- question-relative,
- representation-relative,
- boundary-relative,
- constraint-relative,
- and mechanization-relative where a real execution path matters.

An operator exposure plane is **not** guaranteed merely because:

- a route exists,
- a panel exists,
- a chart exists,
- a surface looks clean,
- a label says "structural",
- or a human says the result is understandable.

One-line distinction:

**An operator exposure plane shows what the seam emitted, not what later layers infer, summarize, or classify about it.**

## 5. Seam -> surface -> plane relationship

This note uses three distinct objects.

### 5.1 Seam

A **seam** is the bounded operator handoff point being inspected.

Examples:

- ingest seam
- clock align seam
- window seam
- transform seam
- compression seam
- anomaly seam
- merge seam
- retention seam
- replay seam

The seam defines:
- what process boundary is active,
- what output object class is being produced,
- and what bounded question is lawful there.

### 5.2 Surface

A **surface** is the broader operator-facing zone associated with the seam.

A surface may contain:
- one primary exposure plane,
- one support plane,
- one derived plane later,
- and shell-local controls outside the active plane.

A surface is broader than a plane.

### 5.3 Plane

A **plane** is one specific exposure geometry or table for one specific output class.

Examples:
- sample table
- time trace
- alignment delta table
- transform heatmap
- spectral line profile
- compression residual plane
- anomaly field plane

One-line summary:

**The seam is the bounded operator boundary.  
The surface is the operator-facing zone around that boundary.  
The plane is the specific exposure object inside that zone.**

## 6. The four operator-plane classes

The current bounded law recognizes four practical operator-plane classes.

---

### 6.1 P0 - Primary exposure plane

#### Definition

The plane that exposes the seam's first-order output directly.

#### Purpose

- expose what the operator actually emitted,
- preserve the seam's own representational constraints,
- make the operator chain inspectable,
- expose first-order output at the lowest lawful exposure layer.

#### Typical examples

- ingest sample table
- ingest sample trace
- clock-align delta table
- transform heatmap
- spectral line profile
- anomaly field plane

#### Main rule

A P0 plane must not be replaced by support, semantic, or review output.

#### One-line summary

**P0 is the first-order plane.**

---

### 6.2 P1 - Support plane

#### Definition

A separate plane or adjacent surface exposing second-order or derived structural/accounting output produced around the seam.

#### Purpose

- expose numeric metadata,
- expose lineage, counts, thresholds, identifiers, timing fields, and derived measurements,
- support human inspection without replacing P0.

#### Typical examples

- sample count table
- window summary table
- alignment stats table
- transform metadata fields
- compression loss table
- support/accounting table

#### Main rule

A P1 plane may support P0.
It may not stand in for P0.

#### One-line summary

**P1 is support and derived exposure, not primary seam exposure.**

---

### 6.3 P2 - Derived inspection plane

#### Definition

A later optional plane that rearranges already-exposed structure into a second visual form for comparison, inspection, or legibility.

#### Purpose

- improve human inspection,
- expose relationships not obvious in the primary plane,
- allow alternate truthful views over the same exposed payload.

#### Typical examples

- transform line profile derived from transform heatmap
- residual plane derived from pre/post compression payloads
- merge adjacency graph
- replay-vs-retained overlay

#### Main rule

A P2 plane must remain derivative of already-exposed P0/P1 output.

It may not become the only visible form of the seam's primary output.

#### One-line summary

**P2 is a later inspection aid, not the first truth of the seam.**

---

### 6.4 P3 - Semantic or review-adjacent surface

#### Definition

A non-primary plane or surface that introduces semantic, evaluative, or readiness language.

#### Rule

This note does not prohibit P3 surfaces globally.

It does prohibit treating P3 as P0.

If semantic or review posture is present, it must remain:
- downstream,
- removable,
- separate,
- and visibly below the structural exposure chain.

#### One-line summary

**P3 may exist later, but never as first-order operator exposure.**

## 7. Operator plane contract rule

Every operator seam that receives a plane contract should define at minimum:

1. **bounded constraint**
2. **primary operator output**
3. **primary output axes and standard units**
4. **support / derived outputs**
5. **support / derived axes and standard units**
6. **most direct exposure form**
7. **more legible alternate forms**
8. **forbidden substitutions**
9. **null / failure posture**
10. **acceptance criteria**

This rule exists so operator planes can be:
- reviewable,
- mechanizable,
- comparable,
- and audited for drift.

One-line summary:

**No operator plane should be built without an explicit plane contract.**

---

## 8. Primary-output rule

The primary output of the seam must be explicit before plane design begins.

Examples:

- ingest seam -> ordered samples / timestamped sample frames
- clock align seam -> aligned timestamps / frame offsets / alignment deltas
- window seam -> bounded windows / frame boundaries
- transform seam -> window-indexed transform-space values
- compression seam -> reduced structure + residual/loss relation
- anomaly seam -> anomaly field / thresholded differences
- merge seam -> merge mapping / grouped region structure
- retention seam -> retained signature / retained support structure
- replay seam -> replay support output / reconstruction-bounded output

The primary output defines the P0 plane.

A plane that does not expose the seam's primary output should not be treated as the primary plane.

---

## 9. Axis and units rule

Every P0 and P1 plane must define its axes and standard units explicitly.

This is required because DME cannot lawfully inspect a plane if its coordinate meaning is drifting.

Examples:

- sample index
- aligned timestamp
- frame index
- frequency bin
- frequency in Hz
- magnitude / normalized magnitude
- delta / offset
- window start / end
- support count
- structural loss percent

Axes may be numeric, tabular, geometric, or relational.

But they must remain explicit.

One-line summary:

**A plane without declared axes is not yet a lawful instrument surface.**

---

## 10. Most direct form vs more legible form

A lawful operator plane must distinguish:

### 10.1 Most direct exposure form

The form that most directly exposes the seam's primary emitted output with the least reshaping.

Examples:

- ingest sample table
- raw bin matrix
- alignment delta table
- pre/post comparison table

### 10.2 More legible alternate form

A later, lawful alternate view that may be easier for humans to inspect while remaining constrained by the same exposed payload.

Examples:

- sample trace over time
- transform heatmap
- spectral profile line plot
- merge adjacency display

### Rule

The more legible form must not erase the more direct form.

If the more legible view becomes the only available view and it flattens distinctions required for inspectability, distortion risk becomes active.

This is especially important under compression and symbolic gravity pressure.

One-line summary:

**Legibility may be added later. Direct exposure must be defined first.**


## 11. Support-plane rule

Support is not prose.

Support is not semantic interpretation.

Support is second-order or adjacent structural/accounting output that helps inspect the primary seam.

Support planes may include:

- numeric fields
- table columns
- counts
- identifiers
- timing fields
- thresholds
- window extents
- declared lens
- source/run ids
- structural-loss values
- alignment deltas
- anomaly magnitudes

Support planes must not include:

- semantic verdicts
- review posture
- memory / identity claims
- readiness posture
- human-helpful summaries standing in for raw output

One-line summary:

**Support is derived structural/accounting exposure, not semantic closure.**

---

## 12. Forbidden substitutions

The following must not substitute for a primary operator exposure plane:

- counts
- object handles
- posture text
- semantic labels
- review labels
- legends that replace direct numeric axes
- dashboards that collapse multiple seams into one pane
- prose explanations
- support/accounting tables by themselves
- visually clean but payload-thin summaries

If any of these become the operator's primary way of understanding the seam, the exposure chain has already weakened.

---

## 13. Null and failure posture

A lawful operator plane must have explicit null and failure posture.

### Null posture

When no payload exists yet, the plane may show:

- blank plane
- neutral geometric null-state
- empty table with axis shell
- explicit zero-payload numeric state

### Failure posture

If payload generation fails or is insufficient, the plane must not replace failure with prose-heavy semantic fallback.

Failure may be shown by:

- absent geometry,
- empty table,
- explicit numeric failure fields outside the plane,
- shell-local status outside the plane.

One-line summary:

**No payload is better than fake payload. Blank is better than semantic disguise.**

---

## 14. Shell relation rule

The shell is not the plane.

The shell may lawfully own:

- source selection
- upload intake
- explicit run control
- active plane selection
- route control
- mode switching
- shell-local failure/readiness status

The shell must not redefine what the plane is exposing.

Shell-local language must remain outside the active primary plane.

If shell posture becomes the operator's primary access path to the seam, the plane has already been demoted.

---

## 15. Mechanization rule for operator planes

An operator plane may be called mechanized only when:

1. the seam's backend output exists,
2. the plane is fed by a real execution path,
3. the rendered plane corresponds to actual seam payload fields,
4. null/failure posture remains explicit,
5. support/derived layers do not overclaim primary exposure,
6. and no stronger semantic claim is made than the payload supports.

Displayed is not mechanized.
A styled chart is not mechanized.
A route with a pane is not mechanized.

One-line summary:

**An operator plane is mechanized only when it truthfully renders real seam output through a real path.**

---

## 16. Exposure-chain rule

The operator-plane family should form an inspectable evidentiary chain.

The target posture is:

- ingest plane
- clock align plane
- window plane
- transform plane
- compression plane
- anomaly plane
- merge plane
- retention plane
- replay plane

The system does **not** need to implement all of these at once.

But any downstream plane should remain accountable to the upstream chain.

This means:

- later planes should not silently replace trust in earlier planes,
- downstream usefulness must not become justification for skipping upstream inspection,
- and operator trust should be built from seam-local exposure, not from final-surface coherence.

One-line summary:

**Trust in the chain should be built bottom-up, one seam at a time.**

---

## 17. Failure modes at the operator-plane level

The following failures should be treated as real:

- primary output absent but support present
- support plane mistaken for primary plane
- more legible view replacing more direct view
- shell posture invading the primary plane
- semantic or review text entering the primary plane
- counts replacing payload
- aggregated display hiding seam-local output
- transform or compression planes showing only summary form
- null/failure fallback replaced by prose

These pair directly with structural exposure failure modes and symbolic gravity notes already active elsewhere.
